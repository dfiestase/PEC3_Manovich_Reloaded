# JupyterLab: medio híbrido para la producción del conocimiento
![enter image description here](https://www.tng-project.org/static/data/lab_logo_tng.png)

## JupyterLab — el cuaderno que convirtió el conocimiento en software

### Introducción

JupyterLab es un entorno de trabajo interactivo utilizado en ámbitos científicos, educativos y culturales para la creación de cuadernos computacionales. Aunque suele presentarse como una herramienta técnica, desde la perspectiva de Lev Manovich puede entenderse como un ejemplo claro de hibridación de medios en la cultura del software. JupyterLab no solo permite escribir código, sino que integra texto, visualización y ejecución en un mismo espacio, transformando la forma en que se produce y comunica el conocimiento.

### "Como funciona JupyterLab"

El entorno de JupyterLab se basa en documentos compuestos por celdas que pueden contener texto narrativo, código ejecutable, imágenes o visualizaciones gráficas. Estas celdas se pueden organizar libremente y ejecutarse de manera independiente, lo que convierte el documento en un sistema dinámico. El usuario no interactúa con un texto cerrado, sino con un entorno vivo que responde a sus acciones en tiempo real.

### Análisis desde las “gafas de Manovich”

Lev Manovich afirma que el software se ha convertido en el medio universal de la cultura contemporánea, reemplazando a tecnologías físicas y electrónicas anteriores y mediando la creación y manipulación de los objetos culturales (*El software toma el mando*). Desde esta perspectiva, JupyterLab puede entenderse como un medio cultural que trasciende su función puramente técnica.

El principio de **hibridación** es central en este caso. Manovich describe cómo, una vez que los medios son simulados en software, sus técnicas comienzan a combinarse, generando nuevas formas culturales. En JupyterLab convergen técnicas propias del texto académico, la programación y la visualización de datos, produciendo un objeto híbrido que no pertenece plenamente a ninguno de estos medios por separado.

La **modularidad**, otro rasgo fundamental de la cultura del software, se manifiesta en la estructura en celdas. Cada celda actúa como un módulo independiente que puede editarse, ejecutarse o reutilizarse sin afectar al conjunto. Este diseño refleja la lógica modular descrita por Manovich, donde los objetos culturales se construyen a partir de unidades discretas y reconfigurables.

Además, JupyterLab ejemplifica el paso del documento estático a la **ejecución dinámica del software**. El contenido no existe como una versión final cerrada, sino como un proceso en constante actualización. De este modo, el conocimiento se presenta como algo legítimo y variable, dependiente de la interacción con el software.

### Reflexión crítica

JupyterLab  no solo muestra cómo el software soporta la cultura, sino que la modela activamente. La producción del conocimiento deja de ser lineal y textual para convertirse en interactiva, modular y visual. En este sentido, JupyterLab confirma la tesis de Manovich: en la cultura digital contemporánea, el software toma el mando y redefine las prácticas culturales que media.


# SkyView App : el cielo convertido en interfaz
![enter image description here](https://cdn.aptoide.com/imgs/6/5/c/65cdc45609666fcbe8e6520b09c3f021_fgraphic.jpg)
### Introducción

SkyView es una aplicación de realidad aumentada que permite identificar estrellas, planetas, constelaciones y satélites simplemente apuntando el dispositivo móvil hacia el cielo. A primera vista, puede parecer una herramienta educativa o divulgativa, ssin embargo, desde la perspectiva de Lev Manovich, SkyView constituye un ejemplo significativo de hibridación contemporánea. La aplicación transforma el cielo en un espacio natural y simbólico, con una interfaz interactiva mediada por software, evidenciando cómo la cultura digital reconfigura nuestra percepción del mundo físico.

### "Como funciona SkyView"

SkyView combina datos astronómicos, sensores del dispositivo (GPS, brújula, giroscopio) y gráficos generados por  un software para superponer información digital sobre el entorno real. La experiencia del usuario depende de múltiples variables: la ubicación geográfica, la hora del día y la orientación del dispositivo. El cielo deja de ser un objeto de contemplación pasiva para convertirse en un espacio navegable e informacional.

La aplicación no muestra una representación fija, sino que actualiza constantemente la información en función del contexto, ofreciendo una visualización dinámica y personalizada del cielo.

### Análisis desde las “gafas de Manovich”

Según Manovich en _El software toma el mando_, el software opera como una capa intermedia que organiza la manera en que accedemos y comprendemos la realidad, estructurando tanto el acceso a la información como la experiencia cultural. SkyView ejemplifica esta idea al convertir el cielo físico en un espacio interpretado a través de bases de datos y algoritmos.

El principio de **hibridación** se manifiesta en fusión de un espacio natural, visualización científica y tecnología de realidad aumentada. SkyView no solo se limita a representar el cielo, sino que integra técnicas de cartografía, simulación y visualización de datos dentro de una interfaz interactiva. Como describe Manovich, cuando las técnicas de distintos medios se combinan dentro del software, emergen nuevas formas culturales híbridas.

La aplicación también responde al principio de **variabilidad**. Manovich sostiene que los objetos culturales digitales no poseen una sola forma única, sino múltiples versiones potenciales. En SkyView, la visualización cambia constantemente según el contexto del usuario, generando una experiencia única e irrepetible en cada uso.

Asimismo, SkyView pone de relieve la **transcodificación cultural**. El conocimiento astronómico, tradicionalmente expresado mediante mapas, textos o diagramas, se traduce aquí a una lógica computacional que lo adapta a la interacción en tiempo real. El cielo se convierte en datos procesables, y los datos en una experiencia visual accesible, demostrando cómo el software reorganiza el conocimiento cultural.

### Reflexión crítica

SkyView ilustra cómo el software redefine nuestra relación con el entorno físico. La observación del cielo deja de ser una práctica contemplativa o simbólica para transformarse en una experiencia guiada por unas interfaces, algoritmos y bases de datos. El usuario ya no mira el cielo directamente, sino a través de una capa digital que filtra, organiza y da sentido a lo que ve. En este sentido, SkyView confirma la tesis de Manovich: el software no solo representa la realidad, sino que la interpreta y la estructura culturalmente.
